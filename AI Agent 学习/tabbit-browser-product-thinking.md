## 背景

Tabbit 背后的团队，是美团的**光年之外**（最早由王慧文老师创办，后并入美团）。

大约一个月前，作者和 Tabbit 产品团队开了场线上交流会，彼时产品还在内测阶段。本文并非评测，而是记录那场对话中关于「浏览器该往何处去」的几点思考。**观点来自光年之外团队。**

## 核心观点

### 一、App 是孤岛，浏览器是大陆

当前数字生活被割裂成两半：手机 vs 电脑。

**移动端的困境：**
- App 之间数据壁垒高（微信 ↔ 飞书 ↔ 钉钉互不相通）
- 每个 App 是一座孤岛，严防数据外流
- AI 在手机上只能做浅层调度，或在单个 App 内部打转

**PC 端浏览器的优势：**
- 承载了绝大多数工作场景（OA、协作文档、资料、邮件、日历……）
- 数据是流动的，DOM 结构是公开的

**核心比喻：**
> Cursor 是程序员的 IDE，那么**浏览器应该是所有白领和知识工作者的 IDE**。

基于此，团队选择基于 Chromium 重做了一个浏览器——只有掌握底座，才能完整、低摩擦地处理用户上下文。

- **插件方式**：依附于页面，权限受限，交互受限
- **浏览器方式**：AI 拥有「上帝视角」，可读取 DOM 结构，理解文档，甚至操作网页按钮

---

### 二、自然语言编译脚本

**问题背景：** 懂技术的人觉得繁琐，不懂技术的人觉得很难（抓视频、转 Markdown、操作 DOM……）

**Tabbit 的解法 Demo：**

用户在侧边栏输入："帮我通过读取 HTML 代码，下载这个页面里的视频。"

→ AI 分析页面 DOM → 识别视频播放器容器 → 找到资源请求地址 → 后台生成 JavaScript 脚本 → 脚本自动运行，视频开始下载。

**本质：工具定义权的下放**

过去：需要等开发者做插件，或自己学 UserScript（油猴脚本），门槛极高。  
现在：自然语言完成编译，AI 充当「翻译官」，把意图翻译成浏览器能执行的代码。

**可实现的能力举例：**
- 屏蔽所有带广告标签的 Div
- 把页面表格数据清洗后导出为 Excel
- 把网页变成深色模式
- 自动化填写重复性表单

> 网页成为了一个可以被捏合、被重塑的泥板。

---

### 三、后台并行的 Agent

Tabbit 支持 Agent 功能，任务运行在独立的标签组里。

**工作方式：**
- 你下达任务（如"调研所有 AI 手机竞品，汇总参数"）
- Agent 在后台独立环境中：搜索 → 打开网页 → 读取 → 汇总
- **主窗口不受打扰**，用户继续写文档、回邮件
- 任务完成后推送通知，交付结果

**设计理念：** 把 Agent 看作一个**不打扰人的后台进程**，回归计算机多任务处理的本质——把耗时、重复的任务挂起，把宝贵的注意力留给需要人类判断的高价值工作。

---

### 四、产品细节

- **毛玻璃效果**：Mac 设计语言变为「毛玻璃」后，特意在 Windows 上也实现了同款视觉效果，保持统一
- **垂直标签页**：解决 Tab 爆炸问题，侧边栏可收起，屏幕利用率提升，信息层级更清晰
